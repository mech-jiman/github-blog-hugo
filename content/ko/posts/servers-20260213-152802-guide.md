## # 개발자라면 꼭 봐야 할 MCP 서버 3가지

어제 저는 놀라운 발견을 했습니다. AI와 데이터를 연결하는 가장 합리적인 방법을 찾았거든요.

**2-3문장 요약:** 이 글에서는 Anthropic이 공개한 'Model Context Protocol(MCP)'와 핵심 저장소인 `servers`를 소개합니다. 복잡한 API 연동 없이 AI가 내 로컬 파일과 데이터베이스를 읽게 만드는 방법을 배울 수 있습니다.

---

## 왜 servers인가요?

🎪 **문제 상황**
많은 개발자들이 겪는 문제가 있습니다. AI에게 내 프로젝트 코드를 분석시키거나, 데이터베이스에 있는 데이터를 기반으로 질문을 하고 싶을 때죠. 하지만 매번 API를 따로 만들고, 텍스트로 복사해서 넣어야 했습니다.

😰 **고민 과정**
저도 처음엔 이걸 몰라서 고생했습니다. 커스텀 플러그인을 직접 만들다가 코드가 꼬이고, 보안 문제까지 걱정하게 되었거든요. "정말 쉬운 방법은 없을까?"라는 질문에 매번 갇혀 있었습니다.

💡 **해결책 발견**
그러다 이 프로젝트를 발견했고, 세상이 바뀌었습니다. 'Model Context Protocol(MCP)'이라는 새로운 표준과, 그 표준을 구현한 사전 구축된 서버 집합인 `servers`를 만난 것이죠.

🎉 **결과와 교훈**
지금은 5분 만에 설정이 끝납니다. AI가 내 시스템의 일부인 것처럼 자연스럽게 대화할 수 있게 되었습니다.

---

## 핵심 기능 TOP 3

이 저장소는 TypeScript로 작성된 다양한 MCP 서버를 포함하고 있습니다. 그중 가장 유용한 3가지를 뽑았습니다.

1.  **`filesystem`** - 로컬 파일 시스템 제어
    *   AI가 허용된 경로의 파일을 읽고, 쓰고, 편집할 수 있게 합니다. 코드 리팩토링이나 문서 생성 시 강력합니다.
2.  **`github`** - 깃허브 리포지토리 통합
    *   복잡한 API 토큰 관리 없이 특정 저장소의 이슈, PR, 파일 내용을 AI가 직접 조회하게 만듭니다.
3.  **`postgres` / `sqlite`** - 데이터베이스 쿼리
    *   직접 SQL을 짜지 않아도, 자연어 요청에 따라 AI가 데이터베이스를 읽고 분석할 수 있습니다.

---

## 설치 및 설정 가이드

가장 인기 있는 `filesystem` 서버를 설치해 볼까요? Node.js가 설치되어 있어야 합니다.

```bash
# 전역 설치
npm install -g @modelcontextprotocol/server-filesystem

# 또는 프로젝트 로컬 설치
npm install @modelcontextprotocol/server-filesystem
```

**💡 자주 발생하는 문제와 해결법:**
- 문제 1: `Command not found` 에러 발생
  → 해결: Node.js 설치 경로를 환경 변수(Path)에 추가했는지 확인하세요.
- 문제 2: 접근 권한 거부 오류
  → 해결: MCP 클라이언트 설정에서 허용할 디렉토리 경로를 정확히 명시했는지 확인하세요.

## 실전 사용 예시

Claude Desktop(클로드 데스크탑) 앱에서 이 서버를 사용하는 설정 예시입니다. `claude_desktop_config.json` 파일을 열어 아래 내용을 추가해 보세요.

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/당신이름/projects"
      ]
    }
  }
}
```

이제 클로드에게 "내 프로젝트 폴더의 구조를 분석해 줘"라고 말해보세요. 놀랍게도 스스로 파일을 열어서 분석해 줄 겁니다.

## 장단점 분석

| 장점 | 단점 |
|------|------|
| ✅ 복잡한 API 없이 표준 프로토콜 사용 가능 | ❌ 아직 생태계가 초기라 문서가 부족할 수 있음 |
| ✅ 타입스크립트로 작성되어 안정적이고 확장 용이 | ❌ 클라이언트(Claude Desktop 등)가 MCP를 지원해야 함 |
| ✅ 이미 유용한 서버(Filesystem, DB 등)가 구현되어 있음 | ❌ 로컬 환경 설정이 익숙하지 않은 초보자에게는 진입장벽 있음 |
